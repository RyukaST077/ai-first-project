# ヒアリング質問集（Phase 3 用）

Phase 2 の抽出ドラフトをユーザに見せた後、**1 メッセージに複数まとめて** 質問する。全部聞く必要はない — ドラフトで明確な部分はスキップし、不明点だけに絞る。

---

## 必須質問（ほぼ毎回聞く）

### Q1. パラメータ（変わる部分）

> この Skill を別プロジェクト / 別対象で使うとき、**変わる情報** は何ですか？
> （例: ファイルパス、テーブル名、ブランチ名、環境名）

抽出用途: SKILL.md 内で `<var>` 形式に抽象化すべき箇所を特定。

### Q2. 起動フレーズ

> 次回この Skill を呼び出すとき、**どんな言葉で声をかけたい** ですか？ 2〜3 個挙げてください。
> （これが SKILL.md の description に入り、Claude がいつ起動するか判断する基準になります）

抽出用途: description の具体化。

### Q3. 除外範囲

> 今回の会話で **Skill には含めたくない** 部分はありますか？
> （例: 途中で失敗して捨てた案、関係ない脱線、特定の試行錯誤）

抽出用途: Skill 本文の絞り込み。

---

## 構造に関する質問

### Q4. 付随ファイルの要否

> 手順の中に **長いコマンドや設定ファイル** がありますか？
> - はい → `scripts/*.sh` や `templates/*` として切り出すことを提案
> - いいえ → SKILL.md 単独で十分

### Q5. 参照ドキュメントの要否

> 細かい失敗対応や FAQ、設計背景など **SKILL.md に書くと冗長になる情報** はありますか？
> - あれば `reference/*.md` に分離

---

## 品質に関する質問

### Q6. 成功判定

> この Skill が「うまく動いた」と **何を見て確認** しますか？
> （成功時のログ・ファイル状態・出力など）

抽出用途: 各 Step の「確認ポイント」記述。

### Q7. 失敗時の対応

> よくある失敗とその対処を思いつく範囲で教えてください。

抽出用途: 「失敗時の対応」セクション。

---

## スコープ確認

### Q8. 既存 Skill との関係

> 近い目的の既存 Skill（`check-endpoints`, `create-snapshot` など）はありますか？ 重複しないか確認したいです。

抽出用途: 重複回避 / 既存 Skill との役割分担。

### Q9. プロジェクト固有 vs 汎用

> この Skill は **このプロジェクト専用** ですか？ それとも他プロジェクトでも使える想定ですか？
> （専用なら `.claude/skills/` で OK。汎用なら `~/.claude/skills/` 配置も検討）

抽出用途: 保存先判断（ただしユーザが既に `.claude/skills/` 指定なら聞かない）。

---

## 「この Skill を使わない」条件

### Q10. 使わない場合

> 逆に、**この Skill を使うべきでない** ケースはありますか？
> （description の「～な場合には使わない」文言に入る。誤起動を減らすために重要）

抽出用途: frontmatter description の精度向上。

---

## 質問テンプレ（コピー用）

```
Phase 2 のドラフトで確認したい点がいくつかあります。まとめて答えてください:

1. パラメータ（再利用時に変わる部分）: <仮説>で合っていますか？他にありますか？
2. 起動フレーズ 2〜3 個: どんな言い回しで呼び出したい？
3. 除外すべき部分: 途中の試行錯誤で Skill に入れないほうがよいものは？
4. 付随ファイル: 長いコマンドや設定を scripts/ に切り出す？
5. 成功判定: 何を見てうまくいったと判断する？
6. 使わないべきケース: この Skill を呼ぶべきでない状況は？
```
