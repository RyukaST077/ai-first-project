# AIを活用したレビューの手順

プログラマーが実装を完了した際、そのまま上司へレビュー依頼をする前に自身でAIとともに自己レビューを行うことでAIを活用した開発でもレビューの負担を最小限に抑えることが可能です。

## 重要な注意点
このレビューコマンドはGitの機能を使ってmainブランチとのdiffを取得し、その内容をレビューするものです。なので、Git未導入である場合やブランチを作成していない場合はうまく機能しないことがあります。

## 手順

### Step1. レビュー観点リストの作成
レビューをする際にどのような点に注目してレビューをするべきか、**レビュー観点一覧を事前に設定**しておきます。それによりプロジェクト固有の観点やこれまでのナレッジを反映したレビューを行うことが可能になります。

`templates\review_list.md` このファイルにレビュー観点を書き込むことで、レビューコマンドを実行した際にそれに応じた観点に沿ってレビューをしてくれます。テンプレートの段階ではサンプルとして記載済みですが、プロジェクトの観点に応じて適宜調整をお願いします。書き方についてはサンプルを参考にしながら上書き保存して下さい。

### Step2. レビューの実施
1. Cline へ *`/review.md`* と送信
2. `docs/reviews/` フォルダ内にレビュー結果が出力されます。

### Step3. レビューを確認し必要があれば修正
レビューは下記の章分けに沿って生成されるので、レビュー結果を確認し、修正が必要だと判断したところを修正して下さい。

````
## Constraints
（例: fetch不可でorigin/mainが古い可能性、テスト未実行など）
## Base
BASE_REF: {origin/main or main}
Compared: ${BASE_REF}...HEAD
## Checklist (from templates/review_list.md)
（RL-xxx: category / item / priority の一覧）
## Summary
（2〜6行）
## Issues（重大度順: critical → high → medium）
- [severity] タイトル（RL-xxx）
  - Checklist: RL-xxx / {category} / {item}
  - Location: file:line-range
  - Evidence:
  - Impact:
  - Recommendation:
  - Validation:
  - Confidence:
## Tests to add
（追加すべきテスト案：優先度つき）
## Questions
（前提不足・判断不能点）
## Verdict
approve | request_changes | comment_only
overall_confidence: 0.0-1.0
````

## レビューをもとに修正する手順
### Step1. まずはAIを使って修正方針を考える
修正する Issue を確定したら下記のプロンプト例を参考に Cline へ Planモードで送信し、修正方針を考えさせる

Issueの 1ブロックをそのまま張り付ける

````
# 依頼
レビューで判明した下記のIssueの修正方針を考えて

---

### [critical] 認証・認可動作の破壊的変更（RL-002, RL-004）
- Checklist: RL-002 / 正しさ / 正常系が仕様どおりに動作する
- Location: backend/src/main/java/jp/co/intec/rwf/config/SecurityConfig.java:78-95
- Evidence: SecurityConfigTestで期待値401が403に変更。認証済みパスへの未認証アクセスが403（Forbidden）ではなく401（Unauthorized）を返すべき設定になっている
- Impact: 既存のフロントエンド実装やAPI仕様書との不整合。認証・認可エラーの区別ができず、適切なユーザー体験を提供できない
- Recommendation: 
  ```java
  // 認証不要パスの設定を見直し
  .requestMatchers("/api/auth/**").permitAll()
  // 認証必要パスでの適切なステータスコード返却
  .requestMatchers("/api/**").authenticated()
  ```
- Validation: SecurityConfigTestの期待値を修正し、認証・認可フローの統合テストを追加
- Confidence: 0.9

---

# 参照ドキュメント
設計書関連： `docs/**/*.md` 
ソースコード： コードベース全体参照可
````

### Step2. AIの修正方針を確認し、問題なければ実装指示

AIとともに修正方針が考えられたら、それをもとに Clineを Act モードに変更し、実装させます。具体的には以下のプロンプト例を参考にして下さい。

