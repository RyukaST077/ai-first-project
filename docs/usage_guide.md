# AI-First Project Template 利用ガイド

このリポジトリは、AI（Clineなど）を活用して効率的に開発を行うためのテンプレートです。
従来のExcel/Word設計書をMarkdownに変換し、AIが理解しやすい形式で管理・実装を進めるフローを提供します。

## 1. ディレクトリ構成

- **`docs/`**: ドキュメント格納場所。AIが生成した設計書やマニュアルなどがここに入ります。
- **`templates/`**: 各種設計書のテンプレート（Excel/Word/Markdown）が格納されています。
- **`.clinerules/`**: Cline用の設定とAIワークフロー定義（`.md`ファイル）が含まれています。
- **`scripts/`**: 設計書変換ツールなどのユーティリティスクリプトがあります。
- **`memory-bank/`**: AI（Cline）がプロジェクトの文脈（仕様、設計、現在の状態）を記憶・共有するためのディレクトリです。

## 2. 環境構築

開発を始める前に、AIエージェントと連携するための環境を整えます。

### 2.1 必須ツールのインストール
このテンプレートのスクリプトを使用するには、以下が必要です。

1.  **Pythonのインストール**
    - `scripts/run.cmd` の実行に必要です。Python 3.12以上を推奨します。
    - Windows環境では `py` ランチャーが使える状態にしてください。
2.  **VSCode & Cline拡張機能**
    - 各開発者のマシンにセットアップしてください。

### 2.2 リポジトリの準備
チーム開発を行う場合、Gitリポジトリ（GitHubなど）を用意し、このテンプレートを展開します。

- **依存ライブラリのインストール**: 初回に `scripts/run.cmd` を実行すると、自動的に仮想環境（`.venv`）が作成され、必要なライブラリがインストールされます。
- **プロキシ設定**: 社内ネットワークなどでプロキシが必要な場合、`scripts/run.cmd` 内の `HTTP_PROXY`, `HTTPS_PROXY` の設定を確認・変更してください。

## 3. 開発ワークフロー

AIと協調した開発は、主に以下の3つのフェーズで進行します。

### Phase 1: 設計 (Design)

既存のExcel/Word設計書をAIが扱えるMarkdown形式に変換・整形します。

1.  **設計書の配置**:
    - リポジトリ直下に **`設計書`** という名前のフォルダを作成し、変換したい `.xlsx` や `.docx` ファイルを配置します。
2.  **Markdown変換**:
    - **`scripts/run.cmd`** を実行します。
    - 変換されたファイルは **`converted_md_design`** フォルダに出力されます。
3.  **AIによる整形**:
    - Clineで以下のコマンドを実行し、変換済みファイルを正式な設計書として取り込みます。
        - コマンド: `/adjust_md_design`
    - 整形された設計書は `docs/design` 配下に保存されます。

### Phase 2: プロジェクト初期化 (Initialization)

AIにプロジェクトの文脈を理解させるため、Memory Bank（AIの記憶）を初期化します。

1.  Clineで以下のコマンドを実行します。
    - コマンド: `/init-memory-bank`
2.  現在の設計書やドキュメントの状態が読み込まれ、`memory-bank/` 配下のファイルが更新されます。

### Phase 3: 実装サイクル (Implementation)

AIエージェントと共に、計画→タスク分解→実装→修正のサイクルを回します。

#### Step 1: 実装計画の作成
機能追加や改修の全体計画を作成します。
- コマンド: `/plan`
- 成果物: `docs/implements_plan.md`
- **ポイント**: まずは共通基盤部分などの実装計画を立てることを推奨します。

#### Step 2: タスク分解と実装
作成された計画に基づき、タスク単位で実装を進めます。

1.  **タスクリスト生成**:
    - 実装計画内のタスクIDを指定してコマンドを実行します。
    - コマンド例: `/tasks.md T-001`
    - `_tasks/` 配下にタスクリストが作成されます。
2.  **実装開始**:
    - タスクリストを指定して実装を指示します。
    - コマンド: `/implement.md @\_tasks/tasklist`
3.  **継続的な対話**:
    - 実装が完了するまで、Clineと対話を続けます。途中で止まった場合は「続きをお願い」と指示します。
    - 完了したら `update memory-bank` と入力して文脈を記憶させ、会話を終了（または次のタスクへ移行）します。

#### Step 3: 修正 (Fix)
エラーや不具合が発生した場合は、修正ガイドに従って対応します。
- コマンド: `/fix`
- 参考: [`docs/fix_guid.md`](./fix_guid.md)

## 4. Tips & その他

### 画面遷移図の取り扱い (Mermaid)
Excel等の図をAIに理解させるため、Mermaid記法に変換して保存することを推奨します。

1.  図をキャプチャし、Copilot等に貼り付けます。
2.  プロンプト: `画像をMermaid記法で出力してほしい`
3.  出力されたコードを `docs/画面遷移図.md` 等として保存します。
