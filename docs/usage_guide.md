# AIファーストプロジェクトテンプレート 利用ガイド

このリポジトリは、AI駆動開発のためのテンプレートとして設計されています。定義されたワークフローと「Memory Bank」の概念を活用することで、文脈を維持し、AIエージェント（Clineなど）から一貫した高品質な成果物を生み出すことができます。

## 1. セットアップと初期化

開発を開始する前に、プロジェクト環境とルールを初期化する必要があります。

### 1.1. ルールの初期化
**コマンド:** `/init-rules`

このワークフローは `docs/` ディレクトリをスキャンし、`.clinerules/02-document.md` に要約を作成します。これにより、AIがどのようなドキュメントが存在するかを理解できるようになります。
*   **前提条件:** ドキュメント類が `docs/` フォルダにあることを確認してください。

### 1.2. Memory Bank の初期化
**コマンド:** `/init-memory-bank`

このワークフローは、AIのコンテキストメモリを初期化します。`docs/` を読み込み、`memory-bank/` 内のファイルに情報を入力します：
*   `projectBrief.md`: プロジェクトの概要とゴール。
*   `productContext.md`: プロダクト要件とコンテキスト。
*   `techContext.md`: 技術的な制約とスタック。
*   `systemPatterns.md`: システムアーキテクチャとパターン。

### 1.3. 環境の初期化
**コマンド:** `/init-env`

プロジェクトの要件に基づいて、ローカル開発環境（例: `npm install`, `pip install`）をセットアップします。これは環境構築のみに焦点を当てており、機能の実装は行いません。

## 2. 設計フェーズ（推奨）

大まかな設計メモや、変換済みのMarkdown設計書がある場合は、このワークフローを使用して構造化します。

### 設計書の整形
**コマンド:** `/adjust_md_design`

このワークフローは、ソースディレクトリ（例: `converted_md_design`）にある非構造化Markdownファイルを受け取り、標準テンプレートを使用して `docs/design` ディレクトリに構造化して配置します。
以下のドキュメントを扱います：
*   基本設計
*   アーキテクチャ
*   データベース設計とテーブル定義
*   画面定義
*   API/外部連携
*   バッチ処理

## 3. 計画フェーズ

設計が整ったら、実装計画を作成します。

### 実装計画の作成
**コマンド:** `/plan.md`

このワークフローは `docs/` を読み込み、包括的な実装計画を `docs/implements_plan.md` に生成します。
計画には以下が含まれます：
*   段階的な実装戦略
*   チームの役割分担（該当する場合）
*   不足している依存関係に対するモック戦略
*   リスク評価

## 4. タスク分解

全体計画を管理可能なタスクに分解します。

### タスクリストの作成
**コマンド:** `/tasks.md [Task-ID]` （例: `/tasks.md T-01`）

このワークフローは、`docs/implements_plan.md` にある大きなタスクを、`_tasks/<TaskID>.md` 内にプルリクエスト（PR）サイズの粒度の細かいサブタスクとして分解します。
*   タスク間の依存関係を考慮します。
*   各PRが単一の検証可能な目的を持つようにします。

## 5. 実装フェーズ

分解されたタスクに基づいて機能を実装します。

### 機能の実装
**コマンド:** `/implement [機能名]` （例: `/implement ログインシステム`）

このワークフローは、特定の機能の実装をガイドします。
1.  機能名を確認します。
2.  `activeContext.md` にPRレベルのタスクリストを作成または更新します。
3.  前提条件と受け入れ基準を確認しなら、ステップバイステップでコードを実装します。
4.  論理的な単位（PR）が完了し検証された場合にのみ、変更をコミットします。

## 6. デバッグと修正

問題が発生した場合は、構造化された修正ワークフローを使用します。

### 問題の修正
**コマンド:** `/fix [問題の説明]`

このワークフローは、デバッグに対して厳格な科学的手法を強制します：
1.  **観測**: 現象を定義します。
2.  **仮説**: 確信度とともに潜在的な原因をリストアップします。
3.  **実験**: 切り分けと検証をステップバイステップで行います。
4.  **解決策**: 修正案を提案し、実装します。
5.  **検証**: 修正を確認し、再発を防止します。

## 7. Memory Bank 管理

`memory-bank/` ディレクトリは、AIの長期記憶です。
*   **常に読む**: AIはタスクの開始時に必ずこれらのファイルを読み、コンテキストを理解します。
*   **更新ルール**:
    *   `activeContext.md`: 現在のタスクの進捗を追跡するために常に更新されます。
    *   `productContext.md`, `systemPatterns.md` など: 主要な設計決定や要件が変更された場合に更新されます。
    *   **トリガー**: 仕様、アーキテクチャ、またはスコープが大幅に変更された場合は、これらのファイルを更新してください。
