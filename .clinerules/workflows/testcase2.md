# テストケース作成指示

## ユーザー入力
このコマンドは `/testcase.md [テスト対象]` の形式で入力されます。
受け取った [テスト対象] をテスト対象の機能としてください。

## 役割
あなたはQAエンジニアとして、手順にしたがってテストケースを作成すること。

## 進行ルール
- 各手順の成果物をユーザーに提示した後、**必ずユーザーの確認を待ってから**次の手順に進むこと
- ユーザーから修正指示があった場合は、その手順の成果物を修正し、再度確認を求めること
- ユーザーが承認した場合のみ、次の手順に進むこと
- 確認時には、現在の手順番号と次に進む手順を明示すること

## 手順

### 1. 要件を読む
まず `docs/` フォルダ内の設計書や関連資料から要件を読み込み、何を実現すべきか、何が正しい動作なのかを把握してください。
この段階では、対象機能の目的、前提条件、制約事項を明確にすることを重視してください。

**【確認】** 要件整理の結果（機能の目的・前提条件・制約事項・正しい動作の定義）をユーザーに提示し、以下の形式で確認を求めてください。
> 📋 **手順1: 要件整理が完了しました。**
> 上記の内容で問題なければ、次の手順（2. 業務フローに分ける）に進みます。
> 修正点があればお知らせください。

### 2. 業務フローに分ける
要件を確認したら、対象機能を業務フロー単位で整理してください。
利用者の操作や処理の流れを分解し、どの場面で何を確認すべきかが見える状態にしてください。

**【確認】** 業務フロー一覧をユーザーに提示し、以下の形式で確認を求めてください。
> 📋 **手順2: 業務フローの整理が完了しました。**
> 上記の内容で問題なければ、次の手順（3. 観点を広げる）に進みます。
> フローの追加・修正・分割などがあればお知らせください。

### 3. 観点を広げる
整理した業務フローをもとに、テスト観点を広げてください。
正常系だけでなく、異常系、境界値、入力条件の違い、分岐パターンなども漏れなく洗い出してください。

**【確認】** テスト観点一覧をユーザーに提示し、以下の形式で確認を求めてください。
> 📋 **手順3: テスト観点の洗い出しが完了しました。**
> 上記の内容で問題なければ、次の手順（4. シナリオ化する）に進みます。
> 観点の追加・削除・修正があればお知らせください。

### 4. シナリオ化する
洗い出した観点を、実際の利用場面がわかる形でシナリオにしてください。
「誰が、どのような条件で、何を行い、どうなるか」が一連の流れとして整理されている状態を目指してください。

**【確認】** テストシナリオ一覧をユーザーに提示し、以下の形式で確認を求めてください。
> 📋 **手順4: テストシナリオの作成が完了しました。**
> 上記の内容で問題なければ、次の手順（5. ケース化する）に進みます。
> シナリオの追加・修正があればお知らせください。

### 5. ケース化する
作成したシナリオを具体的なテストケースに落とし込んでください。
各テストケースには、少なくとも以下を明確に記載してください。

* 入力値
* 操作手順
* 期待結果

**【確認】** テストケース一覧をユーザーに提示し、以下の形式で確認を求めてください。
> 📋 **手順5: テストケースの作成が完了しました。**
> 上記の内容で問題なければ、ファイルに出力します。
> ケースの追加・修正があればお知らせください。

## 出力

- 出力先: `_testcase/testcase_<機能名>.md`
- ファイル名の `<機能名>` はユーザーが指定した機能名を英語のスネークケースに変換して使用すること
- 最終的なテストケース表だけでなく、要件整理、業務フロー、テスト観点、シナリオの中間成果も同じファイルに残すこと
- 中間成果は、テストケースの根拠が追跡できる粒度で簡潔に整理すること
- 出力は以下の構成に従うこと

## 出力構成

```markdown
# テストケース: <機能名>

## 1. 要件整理
- 機能の目的:
- 前提条件:
- 制約事項:
- 正しい動作の定義:

## 2. 業務フロー
| フローNo | 業務フロー | 概要 |
| -------- | ---------- | ---- |
| F-01     |            |      |

## 3. テスト観点一覧
| 観点No | 業務フローNo | テスト観点 | 確認内容 |
| ------ | ------------ | ---------- | -------- |
| VP-01  | F-01         | 正常系     |          |

## 4. テストシナリオ
| シナリオNo | 業務フローNo | 条件/状況 | シナリオ |
| ---------- | ------------ | --------- | -------- |
| SC-01      | F-01         |           |          |

## 5. テストケース
| テストケースNo | 機能ID | 機能名 | テスト観点 | 優先度 | 前提条件 | 入力値 / 操作 | 期待結果 | 判定基準 | 備考 |

| TC-001         |        |        | 正常系     |        |          |               |          |          |      |

```
