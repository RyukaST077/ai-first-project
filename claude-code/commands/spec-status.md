# 仕様ステータス

<background_information>
- **ミッション**: 仕様の状態と進捗を包括的に表示する
- **成功条件**:
  - 現在フェーズと完了状態を示せる
  - 次アクションとブロッカーを特定できる
  - 進捗を明確に可視化できる
</background_information>

<instructions>
## 主タスク
機能 **$1** の全フェーズ進捗を示すステータスレポートを生成する。

## 実行手順

### ステップ1: 仕様コンテキストの読込
- `{{KIRO_DIR}}/specs/$1/spec.json` を読み、メタデータとフェーズ状態を確認
- 既存ファイル `requirements.md`, `design.md`, `tasks.md`（存在する場合）を読む
- `{{KIRO_DIR}}/specs/$1/` ディレクトリ内の利用可能ファイルを確認

### ステップ2: 状態分析

**各フェーズを解析**:
- **要件**: 要件数と受け入れ基準数を集計
- **設計**: アーキテクチャ、コンポーネント、図の有無を確認
- **タスク**: 完了タスク数と総タスク数を集計（`- [x]` と `- [ ]` を解析）
- **承認**: spec.json の承認状態を確認

### ステップ3: レポート生成

spec.json 指定言語で次を含むレポートを作成:
1. **現在フェーズと進捗**: ワークフロー上の位置
2. **完了状況**: フェーズごとの完了率
3. **タスク内訳**: tasks がある場合は完了/残件を表示
4. **次のアクション**: 次に何を行うべきか
5. **ブロッカー**: 進行を妨げる問題

## 重要制約
- 言語は spec.json 指定を使用
- 完了率を正確に算出
- 次アクションは具体的なコマンドで提示
</instructions>

## ツールガイダンス
- **Read**: 最初に spec.json、続いて必要な仕様ファイルを読み込む
- **慎重に解析**: tasks.md のチェックボックスから完了データを抽出
- **Glob** で存在する仕様ファイルを確認

## 出力仕様

spec.json 指定言語でステータスレポートを提供:

**レポート構成**:
1. **機能概要**: 名前、フェーズ、最終更新
2. **フェーズ状態**: 要件 / 設計 / タスク の完了率
3. **タスク進捗**: タスクがあれば X/Y 完了を表示
4. **次アクション**: 次に実行する具体コマンド
5. **問題点**: ブロッカーや不足要素

**形式**: 見やすく、ステータスに絵文字（✅/⏳/❌）を使用

## 安全性とフォールバック

### エラーシナリオ

**仕様が見つからない**:
- **メッセージ**: 「`$1` の仕様が見つかりません。`{{KIRO_DIR}}/specs/` の利用可能な仕様を確認してください」
- **アクション**: 利用可能な仕様ディレクトリを一覧表示

**仕様が不完全**:
- **警告**: どのファイルが不足しているか特定
- **推奨アクション**: 次フェーズのコマンドを提示

### 全仕様一覧

全仕様を確認するには:
- 引数なしで実行、またはワイルドカードを使用
- `{{KIRO_DIR}}/specs/` 内の全仕様と状態を表示
