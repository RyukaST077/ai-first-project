## 概要

今回のガイドでは**新規開発**全体における実装部分にのみ焦点を当て、AIを十分に活用する方法について紹介しています。
設計やテストの部分は今回のガイドの対象外なので従来通りの手順で行います。

```mermaid
flowchart LR
    
    A["1. 要件定義"]
    B["2. 設計"]
    D["4. テスト"]
    E["5. デプロイ"]
    F["6. 運用・保守"]

    subgraph g1["今回のガイドの範囲"]
        C["3. 実装"]
    end
    A --> B
    B --> C
    C --> D
    D --> E
    E --> F

```

## 現在のAIの実力

**基本的にAIが完璧にコーディングしてくれることはありません。**

簡単な、CRUD操作のみのアプリケーションであれば作成は可能ですが複雑なシステムになればなるほど人の手による修正が必要になってきます。

しかし、簡単な部分はAIにAIができなかった部分のみを人間が修正してあげることで実装全体の効率を高めることが期待できます。

## 使用する際の前提条件

1. 既存の設計書があり
1. ソースコードがない状態（サンプルがある場合が多いから全くない状態ではない）
1. スクラッチ開発経験あり
1. AIを使わなくても実装できる人が使用する（誤解を招かない）
1. AIをコーディング効率を高めるためのものとして使う
1. 生成されたコードを説明できるものに収める（まったく何をしているか説明できないコードは使用を控える）
1. 生成されたコードは60点のたたき台である
