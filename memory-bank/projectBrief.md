# **Project Brief**

## **1. プロジェクト概要**

[ターゲットユーザー] のために [アクション: 例: 在庫追跡を効率化する] ための [アプリケーションの種類] を構築します。
このプロジェクトは、[ソリューション] を提供することで [問題] を解決することを目指しています。

## **2. コア要件 (必須事項)**

システムには以下を含める必要があります:

* **認証**: [プロバイダー] を介した安全なログイン。
* **データ可視化**: [ライブラリ] を使用したリアルタイムチャート。
* **エクスポート**: CSV/PDF エクスポート機能。

## **3. スコープ境界**

### **スコープ内 (In-Scope)**

* デスクトップ Web ブラウザ (Chrome, Edge, Firefox)。
* 英語および日本語の UI サポート (i18n)。

### **スコープ外 (Out-of-Scope / 制約)**

* モバイルネイティブアプリなし。
* フェーズ1ではオフラインモードのサポートなし。
* ダークモードなし。

## **4. 技術的な制約と選好**

* **フレームワーク**: Next.js (App Router)。
* **言語**: TypeScript (Strict)。
* **スタイリング**: Tailwind CSS。
* **状態管理**: React Context (必要な場合を除き Redux は避ける)。
* **データベース**: Supabase (PostgreSQL)。

## **5. ターゲットユーザーペルソナ**

* **役割**: 在庫管理者。
* **技術リテラシー**: 低〜中程度。
* **ニーズ**: ハイコントラストな UI、明確なエラーメッセージ、最小限のクリック数。
