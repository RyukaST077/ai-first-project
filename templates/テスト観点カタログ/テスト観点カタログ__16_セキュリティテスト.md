## 共通

### インジェクション

##### 入力値

###### SQL
- **観点**: メタキャラクタが含まれる入力値がSQLの一部に使用される場合、［システム機能設計書の記載通りの挙動］となること。
  - **カウント対象**: SQL数
  - **補足**: ・チェックポイントは「セキュリティテスト_補足」シート ＞ ■SQLインジェクション対策 ＞ No.1 を参照。
- **観点**: 検索条件文字列が含まれる入力値がSQLの一部に使用される場合、［システム機能設計書の記載通りの挙動］となること。
  - **カウント対象**: SQL数
  - **補足**: ・チェックポイントは「セキュリティテスト_補足」シート ＞ ■SQLインジェクション対策 ＞ No.2 を参照。

###### OSコマンド
- **観点**: OSコマンドを起動できる処理の入力値にOSコマンドが含まれる場合、［システム機能設計書の記載通りの挙動］となること。
  - **カウント対象**: OSコマンド起動処理数

###### メールヘッダ
- **観点**: 入力値がメールヘッダに使用される場合、入力された改行コードと宛先を追加するヘッダ文字列が［システム機能設計書の記載通り］に処理されること。
  - **カウント対象**: メール送信、転送処理数
- **観点**: 入力値がメールヘッダに使用される場合、本文を改竄するコードが［システム機能設計書の記載通り］に処理されること。
  - **カウント対象**: メール送信、転送処理数

###### XML
- **観点**: 入力値がXMLに使用される場合、入力された特殊文字が［システム機能設計書の記載通り］に処理されること。
  - **カウント対象**: ファイル種類数

### ディレクトリ・トラバーサル

##### 入力値
- **観点**: 相対パスが含まれる入力値がファイルやディレクトリのアクセスに使用される場合、不正なアクセスができないこと。
  - **カウント対象**: ファイルアクセス処理数

### 認可制御

#### ユーザー認証
- **観点**: 認証処理がサーバー側で行われていること。
  - **カウント対象**: ユーザー認証処理数
- **観点**: ユーザー認証情報が推測できる状態となっていないこと。
  - **カウント対象**: アプリケーション数
- **観点**: 権限のないユーザーによって、ユーザ認証情報の上書きが可能な状態となっていないこと。
  - **カウント対象**: アプリケーション数

###### URL直接アクセス
- **観点**: ユーザー認証が必要な機能に対し、未認証の状態でURLを直接指定してアクセスした場合、ユーザー認証を求める処理に遷移すること。
  - **カウント対象**: 機能数
- **観点**: ユーザー認証が必要な機能に対し、認証済の状態でURLを直接指定してアクセスした場合、ユーザー認証が有効となっていること。
  - **カウント対象**: 機能数

#### ユーザー権限
- **観点**: ユーザーが権限のある対象にアクセスできること。
  - **カウント対象**: ユーザー権限数
  - **補足**: ・認証が必要な画面やウェブサービスでは、認証済みユーザーのみアクセスできるか検証する。
- **観点**: ユーザーが権限のない対象にアクセスできないこと。
  - **カウント対象**: ユーザー権限数
- **観点**: ユーザーが権限のある情報を表示できること。
  - **カウント対象**: ユーザー権限数
- **観点**: ユーザーが権限のない情報を表示できないこと。
  - **カウント対象**: ユーザー権限数
- **観点**: 異なるアカウントから同時にアクセスした場合、［セキュリティ要件定義/方式設計書の記載通りの挙動］となること。
  - **カウント対象**: アプリケーション数
- **観点**: 同じアカウントから同時にアクセスした場合、［セキュリティ要件定義/方式設計書の記載通りの挙動］となること。
  - **カウント対象**: アプリケーション数
  - **補足**: ・ウェブアプリケーションの場合は、別ウィンドウでアクセスした場合を検証する。
- **観点**: URLパラメータの変更により、対象ユーザーの権限を超えたアクセス、情報表示が行われないこと。
  - **カウント対象**: アプリケーション数

### セッション管理

###### セッションID
- **観点**: 独自実装したセッション管理機構を使用していないこと。
  - **カウント対象**: アプリケーション数
- **観点**: セッションの有効期限に設定した時間を超えた場合、［方式設計書の記載通りの挙動］となること。
  - **カウント対象**: アプリケーション数

### クロスサイト・リクエスト・フォージェリ（CSRF）
- **観点**: DB更新などを行う機能に対し、外部から不正なパラメータが入力された場合、［画面遷移図/システム機能設計書の記載通りの挙動］となること。
  - **カウント対象**: 機能数

### クロスサイト・リクエスト・フォージェリ（CSRF）
- **観点**: DB更新などの処理がトークンにより制御されていること。
  - **カウント対象**: 機能数

### クロスサイト・スクリプティング

##### 入力値

###### URL
- **観点**: 入力値を使用しURLを出力する場合、XSS対策により、JavaScript等として不正な処理が実行されないこと。
  - **カウント対象**: 機能数
  - **補足**: ・チェックポイントは「セキュリティテスト_補足」シート ＞ ■XSS対策 ＞ No.1 を参照。

### その他

##### 環境
- **観点**: 対象の環境にあるソフトウェアがすべて更新され、脆弱性を含むバージョンとなっていないこと。
  - **カウント対象**: システム数
- **観点**: 脆弱性が検知されているコンポーネントやライブラリを使用していないこと。
  - **カウント対象**: システム数
- **観点**: フレームワークやライブラリのセキュリティ対策を使用することにより［方式設計書の記載通りの挙動］となること。
  - **カウント対象**: システム数

###### ID/パスワード
- **観点**: アプリケーションが使用するID/パスワードが別ファイルに定義してあり、その別ファイルはセキュアな状態で保管されていること。
  - **カウント対象**: システム数

##### データ
- **観点**: 機密性の高い情報が平文の状態で長期間保存されないこと。
  - **カウント対象**: 機密性の高い項目数
  - **補足**: ・例：個人情報、ユーザー認証情報、クレジットカード番号
- **観点**: 機密性の高い情報が平文の状態で内外へ送信されないこと。
  - **カウント対象**: 機密性の高い項目数
- **観点**: 暗号化鍵の強度が［方式設計書の記載通り］であること。
  - **カウント対象**: アプリケーション数
- **観点**: 暗号化鍵のローテーションが［方式設計書の記載通り］であること。
  - **カウント対象**: アプリケーション数

#### ファイル取込
- **観点**: ウィルスに感染したファイルがアップロードされた場合、サーバで検知してブロックされること。
  - **カウント対象**: ファイル取込処理数
  - **補足**: ・テスト用にウィルスに感染したファイルを使用する。
- **観点**: ウィルスに感染したファイルがアップロードされた場合の挙動が［方式設計書の記載通り］であること。
  - **カウント対象**: ファイル取込処理数
  - **補足**: ・テスト用にウィルスに感染したファイルを使用する。

#### ログ出力

##### 出力内容
- **観点**: 機能の実行状況がログとして出力され、追跡できる状態になっていること。
  - **カウント対象**: 機能数
- **観点**: アプリケーションへのアクセス状況がログとして出力され、追跡できる状態になっていること。
  - **カウント対象**: アプリケーション数

###### エラーメッセージ
- **観点**: ユーザー認証のエラーメッセージで、ユーザーID、パスワードの誤りであることが読み取れる内容となっていないこと。
  - **カウント対象**: ユーザー認証処理数
- **観点**: スタックトレースを出力していないこと。
  - **カウント対象**: アプリケーション数

## ウェブアプリケーション

### インジェクション

##### 入力値

###### HTTPレスポンスヘッダ
- **観点**: 入力値がHTTPレスポンスヘッダに使用される場合、入力された改行コードとCookieを追加するコードが［システム機能設計書の記載通り］に処理されること。
  - **カウント対象**: 画面数
- **観点**: 入力値がHTTPレスポンスヘッダに使用される場合、リダイレクト先URLを書き換えるコードが［システム機能設計書の記載通り］に処理されること。
  - **カウント対象**: 画面数

### 認可制御

#### ユーザー認証

###### タイムアウト
- **観点**: ログインから一定の時間が経過した場合、再度ユーザー認証を行うこと。
  - **カウント対象**: 画面数
- **観点**: 画面の表示から一定の時間が経過した場合、再度ユーザー認証を行うこと。
  - **カウント対象**: 画面数

##### WebSocket
- **観点**: ユーザー認証が必要な場合、ユーザー認証をしていなければ接続できないこと。
  - **カウント対象**: 画面数

#### ユーザー権限
- **観点**: hidden項目の変更により、対象ユーザーの権限を超えたアクセス、情報表示が行われないこと。
  - **カウント対象**: 画面数
- **観点**: Cookieの変更により、対象ユーザーの権限を超えたアクセス、情報表示が行われないこと。
  - **カウント対象**: 画面数

### セッション管理

###### セッションID
- **観点**: ユーザー認証（ログイン）の前後でセッションIDが変化すること。
  - **カウント対象**: ユーザー認証処理数
- **観点**: SSLを使用する場合、セッションIDを保持するCookieにセキュア属性が付与されていること。
  - **カウント対象**: システム数
- **観点**: Cookieをオフにしてアクセスした場合、セッションIDがURL埋め込みにならないこと。
  - **カウント対象**: システム数
- **観点**: セッションIDをURL埋め込みにしている場合、外部リンクからReferer経由でセッションIDが漏洩しないこと。
  - **カウント対象**: アプリケーション数

### クロスサイト・リクエスト・フォージェリ（CSRF）
- **観点**: DB更新などの処理実行前に、再度ユーザー認証を求める画面へ遷移すること。
  - **カウント対象**: 機能数

### クロスサイト・スクリプティング

##### 入力値

###### メタキャラクタ
- **観点**: 画面からのユーザー入力やDBから取得した値を画面に表示する場合、メタキャラクタがサニタイジングされること。
  - **カウント対象**: 入力項目数
  - **補足**: ・チェックポイントは「セキュリティテスト_補足」シート ＞ ■XSS対策 ＞ No.2 を参照。

### クロスサイト・スクリプティング

###### スタイルシート
- **観点**: 任意のサイトからスタイルシートを読み込んでいないこと。
  - **カウント対象**: 画面数

###### Content-Type
- **観点**: Content-Typeが[方式設計書の記載通り]でない場合、エラーとなること。
  - **カウント対象**: 画面数

##### 出力内容

###### JavaScript
- **観点**: JavaScript内に動的項目が出力されていないこと。
  - **カウント対象**: 画面数
  - **補足**: ・チェックポイントは「セキュリティテスト_補足」シート ＞ ■XSS対策 ＞ No.3 を参照。

###### HTML
- **観点**: HTMLの属性値が「"」か「'」で囲んで出力されること。
  - **カウント対象**: 画面数

###### HTTPレスポンスヘッダ
- **観点**: HTTPレスポンスヘッダに「X-Content-Type-Options」を指定することにより、レスポンスの内容をもとにコンテンツの種類を自動判定するIEの機能が動作しないこと。
  - **カウント対象**: 画面数
  - **補足**: ・チェックポイントは「セキュリティテスト_補足」シート ＞ ■XSS対策 ＞ No.4 を参照。
- **観点**: HTTPレスポンスヘッダに「X-XSS-Protection」を指定することにより、ブラウザのXSSフィルタが有効となること。
  - **カウント対象**: 画面数
  - **補足**: ・チェックポイントは「セキュリティテスト_補足」シート ＞ ■XSS対策 ＞ No.5 を参照。
- **観点**: HTTPレスポンスヘッダに「(X-)Content-Security-Policy」を指定することにより、Javascript等のリソースの取得元が制限されること。
  - **カウント対象**: 画面数
  - **補足**: ・チェックポイントは「セキュリティテスト_補足」シート ＞ ■XSS対策 ＞ No.6 を参照。

###### Cookie
- **観点**: 認証に使用するCookieに対しHttpOnly属性を付与することにより、JavascriptでCookieが取得されないこと。
  - **カウント対象**: システム数
  - **補足**: ・チェックポイントは「セキュリティテスト_補足」シート ＞ ■XSS対策 ＞ No.7 を参照。

### クリックジャッキング

###### HTTPレスポンスヘッダ
- **観点**: HTTPレスポンスヘッダに「X-FRAME-OPTION」を指定することにより、frame要素やiframe要素によるページ読み込みが制限されること。
  - **カウント対象**: 画面数

### 踏み台攻撃

##### 入力値
- **観点**: リダイレクトやフォワードを行う処理について、URLをパラメータで保持する場合、ホワイトリストによる検証で許可されたURLのみが使用されること。
  - **カウント対象**: 機能数
- **観点**: URLを含むパラメータの値を意図的に変更した場合、［画面遷移図/方式設計書の記載通りの挙動］となること。
  - **カウント対象**: 機能数

### ファイル公開範囲
- **観点**: ファイルの公開範囲が［方式設計書の記載通り］であること。
  - **カウント対象**: ファイル種類数

### その他

  - **カウント対象**: ブラウザ種類数
