## バリデーション

### 単項目バリデーション

#### 必須バリデーション

##### 必須項目
- **観点**: 必須項目が未入力の場合、エラーとなること。
  - **カウント対象**: 必須項目数

##### 任意項目
- **観点**: 必須項目が入力され、任意項目が未入力の場合、エラーとならないこと。
  - **カウント対象**: 任意項目数

#### 文字列長バリデーション
- **観点**: 文字列項目に最大長が入力された場合、エラーとならないこと。
  - **カウント対象**: 文字列項目数
  - **補足**: ・記号文字を含むパターンも検証する。
- **観点**: 文字列項目に最大長＋1が入力された場合、エラーとなること。
  - **カウント対象**: 文字列項目数
- **観点**: 文字列項目に最小長が入力された場合、エラーとならないこと。
  - **カウント対象**: 文字列項目数
- **観点**: 文字列項目に最小長－1が入力された場合、エラーとなること。
  - **カウント対象**: 文字列項目数
- **観点**: サロゲートペア文字を許容する設計となっている場合、サロゲートペア文字を含んだ値が入力された結果がエラーとならないこと。
  - **カウント対象**: 文字列項目数
- **観点**: 特殊文字（結合文字や合字など）を許容する設計となっている場合、文字列長が適切に判定されること。
  - **カウント対象**: 文字列項目数
  - **補足**: ・異体字セレクタやZWJで結合された絵文字など

#### 数値バリデーション

##### 桁数
- **観点**: 数値項目の整数部に最大桁、小数部に最大桁が入力された場合、エラーとならないこと。
  - **カウント対象**: 数値項目数
- **観点**: 数値項目の整数部に最大桁＋1、小数部に最大桁が入力された場合、エラーとなること。
  - **カウント対象**: 数値項目数
- **観点**: 数値項目の整数部に最大桁、小数部に最大桁＋１が入力された場合、エラーとなること。
  - **カウント対象**: 数値項目数

##### 有効範囲
- **観点**: 数値項目に有効範囲の上限値が入力された場合、エラーとならないこと。
  - **カウント対象**: 数値項目数
- **観点**: 数値項目に有効範囲の上限値＋1が入力された場合、エラーとなること。
  - **カウント対象**: 数値項目数
- **観点**: 数値項目に有効範囲の下限値が入力された場合、エラーとならないこと。
  - **カウント対象**: 数値項目数
- **観点**: 数値項目に有効範囲の下限値－1が入力された場合、エラーとなること。
  - **カウント対象**: 数値項目数
- **観点**: 数値項目に数値以外が入力された場合、エラーとなること。
  - **カウント対象**: 数値項目数

#### 文字種バリデーション
- **観点**: 許容対象の文字が入力された場合、エラーとならないこと。
  - **カウント対象**: テキスト入力項目数
- **観点**: 許容対象外の文字が入力された場合、エラーとなること。
  - **カウント対象**: テキスト入力項目数

#### その他のバリデーション

##### コード値
- **観点**: コード値を入力値とする項目について、有効範囲内の値が入力された場合、エラーとならないこと。
  - **カウント対象**: コード値項目数
- **観点**: コード値を入力値とする項目について、有効範囲外の値が入力された場合、エラーとなること。
  - **カウント対象**: コード値項目数
  - **補足**: ・通常の画面操作では入力できない値（ラジオボタンやプルダウンの値）が入力された場合の挙動も検証する。

##### 日付
- **観点**: 日付項目に有効な書式の値が入力された場合、エラーとならないこと。
  - **カウント対象**: 日付項目数
- **観点**: 日付項目に無効な書式の値が入力された場合、エラーとなること。
  - **カウント対象**: 日付項目数
- **観点**: 日付項目に実在日が入力された場合、エラーとならないこと。
  - **カウント対象**: 日付項目数
  - **補足**: ・業務要件に即した日付、時間の最小単位で処理されることも検証する。
- **観点**: 日付項目に実在日以外が入力された場合、エラーとなること。
  - **カウント対象**: 日付項目数
  - **補足**: ・例：2017/1/32

##### メールアドレス
- **観点**: メールアドレス項目に有効な値が入力された場合、エラーとならないこと。
  - **カウント対象**: メールアドレス項目数
- **観点**: メールアドレス項目に無効な値が入力された場合、エラーとなること。
  - **カウント対象**: メールアドレス項目数

##### アプリケーション固有のバリデーション
- **観点**: アプリケーション固有のバリデーションがある場合、バリデーションの結果が［システム機能設計書の記載通り］であること。
  - **カウント対象**: 固有バリデーション数

### 相関バリデーション
- **観点**: 特定の条件により必須・任意が変わる項目について、該当項目が必須となっている状態で、値が未入力の場合、エラーとなること。
  - **カウント対象**: 条件付き項目数
- **観点**: 特定の条件により必須・任意が変わる項目について、該当項目が任意となっている状態で、値が未入力の場合、エラーとならないこと。
  - **カウント対象**: 条件付き項目数
- **観点**: 相関バリデーション対象項目について、有効な値の組み合わせが各項目に入力された場合、エラーとならないこと。
  - **カウント対象**: データパターン数
- **観点**: 相関バリデーション対象項目について、無効な値の組み合わせが各項目に入力された場合、エラーとなること。
  - **カウント対象**: データパターン数

#### DBとの相関バリデーション
- **観点**: 有効なレコードが存在する場合、エラーとならないこと。
  - **カウント対象**: データパターン数
  - **補足**: ・ここでの「有効なレコード」は、例えば認証（入力されたIDをDBのレコードと照合する）のような機能で、結果が「正」となるレコードを指す。
- **観点**: 有効なレコードが存在しない場合、エラーとなること。
  - **カウント対象**: データパターン数

## データベースアクセス

### DB操作

#### 共通

##### 検索/更新/削除条件

###### 単一条件
- **観点**: 有効な値が条件に指定された場合、期待したレコードが取得、更新、削除されること。
  - **カウント対象**: SQL数
- **観点**: 無効な値が条件に指定された場合、レコードが取得、更新、削除されないこと。
  - **カウント対象**: SQL数
- **観点**: 値が条件に指定されなかった場合、［システム機能設計書/共通コンポーネント設計書の記載通りの挙動］となること。
  - **カウント対象**: SQL数

###### 複数条件
- **観点**: 有効な値の組み合わせが条件に指定された場合、期待したレコードが取得、更新、削除されること。
  - **カウント対象**: データパターン数
- **観点**: 無効な値の組み合わせが条件に指定された場合、レコードが取得、更新、削除されないこと。
  - **カウント対象**: データパターン数
- **観点**: 値が条件に指定されなかった場合、［システム機能設計書/共通コンポーネント設計書の記載通りの挙動］となること。
  - **カウント対象**: SQL数

- **観点**: 完全一致する値が指定された場合、期待したレコードが取得、更新、削除されること。
  - **カウント対象**: SQL数
- **観点**: 完全一致しない値が指定された場合、レコードが取得、更新、削除されないこと。
  - **カウント対象**: SQL数

###### 部分一致
- **観点**: 前方一致する値が指定された場合、期待したレコードが取得、更新、削除されること。
  - **カウント対象**: SQL数
- **観点**: 前方一致しない値が指定された場合、レコードが取得、更新、削除されないこと。

- **観点**: 後方一致しない値が指定された場合、レコードが取得、更新、削除されないこと。

- **観点**: 中間一致する値が指定された場合、期待したレコードが取得、更新、削除されること。

- **観点**: 中間一致しない値が指定された場合、レコードが取得、更新、削除されないこと。
  - **カウント対象**: SQL数

###### 抽出範囲
- **観点**: 月、年度の切り替えにより抽出対象が変わる場合、取得、更新、削除されたレコードが［システム機能設計書/共通コンポーネント設計書の記載通り］であること。
  - **カウント対象**: データパターン数
  - **補足**: ・月、年度の切り替えに影響される抽出条件の例

###### エスケープ
- **観点**: 条件にワイルドカード文字が指定された場合、該当の文字が含まれるレコードが取得、更新、削除されること。

  - **補足**: ・ワイルドカード文字例：「%」「_」「％」「＿」

###### 件数上限
- **観点**: 処理件数に上限がある場合、上限の件数以上のレコードが取得、更新、削除されないこと。
  - **カウント対象**: SQL数

##### 登録/更新内容
- **観点**: 全項目に値が指定された場合、レコードが登録されること。
  - **カウント対象**: 登録SQL数
- **観点**: 更新対象の項目すべてに値が指定された場合、レコードが更新されること。
  - **カウント対象**: 更新SQL数
- **観点**: 必須項目に値が指定されていない場合、レコードが登録されないこと。
  - **カウント対象**: 登録SQL数
- **観点**: 更新対象の必須項目に値が指定されていない場合、レコードが更新されないこと。
  - **カウント対象**: 更新SQL数
- **観点**: 必須項目は値あり、任意項目は値なしの場合、レコードが登録、更新されること。
  - **カウント対象**: 登録/更新SQL数
